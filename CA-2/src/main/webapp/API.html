Test af vores REST API
Vi tester vores REST API metoder ved at udføre REST assured test på dem.
For at kunne gøre det har vi indsat nogle plugins i vores pom fil samt
lavet en @Beforeclass i vores testklasse. Metoden setUpBeforeAll() sørger
for at vores baseURI, port, basePath og defaultParser er i orden.
Vi sætter baseURI til "http://localhost" som er det vores URL starter
med når vi kører programmet. Port nummeret bliver sat til 8080 som bliver
brugt som standard. basePath bliver sat til "/CA-2" som er navnet på vores
projekt. DefaultParser bliver sat til Json som er det vi bruger til at lave data om.

Den første test vi udfører, er for at se om vores server er startet og kører.
Testen serverIsRunning() tjekker om vores server kører, så hvis vi har
startet serveren først vil testen give rigtigt. Når serveren kører kan
vi udføre test på vores API metoder. Vi tester vores getPerson metode
på 3 forskellige måder. Den første test som bare hedder getPerson() ser om
den person vi henter ud gennem vores API på et givent ID stemmer overens
med den person med det samme ID i databasen.
Udover at tjekke om vores API metode virker kan vi teste på hvilke fejl vi
får, hvis der bliver indtastet noget forkert i forhold til vores API.
Test metoderne getPersonWrongArguments og getPersonWrongRoute ser hvordan
vores program håndterer hvis der bliver givet forkerte argumenter eller
rute til vores API metoder.
