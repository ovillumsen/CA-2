Hvordan man bruger programmet:
Som vores program er bygget op forhindrer vores REST assured test at vi starter
programmet så længe serveren ikke kører. Da serveren ikke kører første gang man
starter projektet er det derfor nødvendigt at gå ind i ServiceIntegrationTest
filen og udkommentere alle vores test. Efter at have gjort det kan man starte
projektet og dermed få startet serveren. Når det er gjort skal de test der blev
udkemmenteret sættes ind igen. Nu kører både serveren og projektet og det er
dermed muligt at teste vores API metoder ved at skrive op i vores URL.
Der er ikke blevet indført så mange API metoder der virker, men det er muligt at,
få en person fra databasen, få alle personer fra databasen og tilføje en person til databasen.
For at få 1 person op fra databasen skal der stå "/CA-2/api/person/{id}",
hvor id skal erstattes med det id på personen man vil have.
For at få alle personer fra databasen skal der stå "/CA-2/api/person/all".
For at tilføje en person til databasen skal der stå "/CA-2/api/person/add".
Udover at kunne teste det ud fra vores URL er der også lavet
almindelige Unit Test og REST Assured Test.Unit Test kan findes i
filen PersonFacadeTest og REST Assured test kan findes i ServiceIntegrationTest.
